STAGE 2: CANDIDATE SELECTION (export_stress_quantum_configs.py)
──────────────────────────────────────────────────────────────
  Input:   Stage 1 JSON manifest + .npz artifacts
  Process: Rank 36 results by loss/complexity/geometry
  Logic:   score = 0.3*loss + 0.3*complexity - 0.4*geometry
  Select:  Top-K (default K=4)
  Output:
    ├─ {dataset}_final_quantum_configs.json
    └─ exported_opt_params/*.npz            (standardized params)
  
  Ranking ensures generalization: low loss & low complexity

STAGE 3: FINALIST CONFIRMATION (run_inter_circles_stress_test.py)
──────────────────────────────────────────────────────────────
  Input:   Top-K candidates from Stage 2
  Process: Re-train each as single-config run (full data)
  Goal:    Verify top performers maintain low loss
  GPU:     ~30 min for K=4
  Output:  finalist_opt_params/*.npz + results.json

KEY ALGORITHMS:
═══════════════════════════════════════════════════════════════════════════════

1. DATA ENCODING:
   Input: Raw data (1000 × d) → PCA (1000 × pcs) → Scale [-0.9, 0.9]
   
2. PQC ENCODING:
   Chebyshev basis: x → T_k(x*α) gates on quantum circuit
   
3. QUANTUM KERNEL:
   Unitary U → Kernel K via outer_kernel (RBF with gamma)
   Projection: Low-rank Nystrom approximation (approx_rank=200)
   
4. LOSS FUNCTION:
   L(θ) = ||K(θ) - I|| / n  (Frobenius norm)
   Goal: Make quantum kernel close to identity
   
5. GRADIENT ESTIMATION:
   SPSA: Single perturbation δ for full gradient estimate
   Efficiency: 1 sample vs 2p for finite differences
   
6. OPTIMIZATION:
   torch_adam + SPSA: 100 iterations, adaptive epsilon (1e-4 → 1e-2)
   
7. RANKING:
   score = 0.30*loss + 0.30*complexity - 0.40*geometry
   Select top-K by rank score
